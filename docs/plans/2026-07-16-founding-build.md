# FetchMux Founding Build Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a local-first, provider-neutral web retrieval router with auditable policy selection, hard budget/deadline controls, BYOK provider adapters, REST and MCP access, a reproducible benchmark runner, and a conversion-oriented launch site.

**Architecture:** A framework-independent TypeScript core owns schemas, ranking, health state, budgets, failover, and route receipts. Provider packages translate only documented HTTP contracts; application packages expose the core through Fastify, MCP stdio, a typed SDK, a benchmark CLI, and a static React site. Customer provider credentials remain in environment/injected resolvers, and hard dollar budgets exclude any provider whose pre-request cost is unknown.

**Tech Stack:** Node.js 24, TypeScript 7, npm workspaces, Zod 4, Vitest 4, Fastify 5, Model Context Protocol SDK 1.x, React 19, Vite 8, Biome 2, Docker.

---

## Execution rules

- Use strict red-green-refactor for every production behaviour. Run each named test before implementation and confirm it fails for the intended missing behaviour.
- Configuration, generated lockfiles, prose, and purely visual CSS are the only test-first exceptions for this founding build.
- Do not make live provider calls in the automated suite. Inject `fetch` and use recorded minimal response shapes derived from official documentation.
- Never log provider credentials or full provider response bodies.
- Do not commit `.env`, benchmark result payloads containing provider content, or secrets.
- Commit after each task only after the task's exact test and `git diff --check` succeed.

**Step 3: Write and fail router tests**

Required cases:

- returns first provider results and a complete route receipt;
- retries a second provider after a retryable 429/5xx/timeout;
- does not retry 400/401/403 or invalid provider responses;
- never attempts fallback when cumulative reserved cost would exceed `maxCostUsd`;
- never starts an attempt after the absolute deadline;
- aborts an in-flight provider at the remaining deadline;
- skips open circuits;
- preserves result provider and rank while removing undefined fields;
- returns safe errors without credential or raw-body fragments;
- IDs and time are deterministic under injected factories.

Example budget case:

```typescript
it("does not spend past the fallback budget", async () => {
  const first = failingProvider("first", { cost: 0.006, retryable: true });
  const second = successfulProvider("second", { cost: 0.006 });
  const router = createRouter([first, second]);

  await expect(router.search(request({ maxCostUsd: 0.01 }))).rejects.toMatchObject({
    code: "BUDGET_EXHAUSTED",
  });
  expect(second.calls).toBe(0);
});
```

**Step 4: Implement the router**

Record every attempt, and include fallback reason codes in the successful receipt. When all attempts fail, expose a FetchMux error code and trace ID, not raw upstream data.

## External-action boundary

The local product, documentation, benchmark harness, sales collateral, and deployment artifact may be completed autonomously. The following require the owner's explicit approval because they spend money, create legal obligations, or act under the owner's identity:

- register or buy a domain, trademark, or company;
- accept provider/commercial terms or create paid provider accounts;
- contact prospects or providers;
- publish a site, repository, package, benchmark, or social post;
- collect real lead/customer data;
- charge a card or accept payment;
- deploy to paid infrastructure.
