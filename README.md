# FetchMux

FetchMux is a provider-neutral retrieval router for AI agents. Clients make one search request and receive normalized evidence from the provider that best fits the requested quality, cost, latency, and freshness policy.

The initial product is deliberately narrow: web retrieval through customer-owned provider accounts. It is not a generic proxy, a credential reseller, or a claim that every API is interchangeable.

## Product promise

> One retrieval API. The right provider for every request.

## Status

Founding build. The name is provisional pending formal domain, trademark, and company-name clearance. No provider partnership or resale right is implied.
